GenUI — Graph Color Fix: Live Preview

Root cause: assignDataColorMapping forces gray.moderate for any single-series chart. Below are all three states side-by-side across Bar, Line, and Area charts.

Current — Broken
Single-series charts render in grey

When GenUI renders a chart with one data key, assignDataColorMapping overrides the color to DEFAULT_COLOR = 'data.background.categorical.gray.moderate' (resolved: hsla(200, 10%, 94%) in light mode — barely visible against white). Multi-series charts work fine because the override only fires when Object.keys(dataColorMapping).length === 1.

Option A — Global Fix
Change DEFAULT_COLOR in tokens.ts
Scope: all charts globally Risk: medium — affects all consumers
// packages/blade/src/components/Charts/utils/tokens.ts - const DEFAULT_COLOR = 'data.background.categorical.gray.moderate'; + const DEFAULT_COLOR = 'data.background.categorical.blue.moderate';

Changing the token from gray.moderate → blue.moderate (hsla(217,100%,73%) in light mode) makes every single-series chart render in Blade blue by default. This also aligns with the categorical sequence order where blue is index 0.

Option B — Scoped Fix (GenUI only)
Pass explicit color prop to chart children in GenUIComponents.web.tsx
Scope: GenUI only Risk: low — no impact on Blade chart consumers
// packages/blade/src/components/GenUI/GenUIComponents.web.tsx {keysToPlot.map((key, index) => ( - <ChartBar key={key} dataKey={key} name={key} /> + <ChartBar key={key} dataKey={key} name={key} color={GENUI_CHART_COLORS[index % GENUI_CHART_COLORS.length]} /> ))} // Same for ChartLine and ChartArea // Add at top of file: + const GENUI_CHART_COLORS = [ + 'data.background.categorical.blue.moderate', + 'data.background.categorical.green.moderate', + 'data.background.categorical.gold.moderate', + 'data.background.categorical.purple.moderate', + 'data.background.categorical.orange.moderate', + ] as const;

Passing an explicit color to each chart child bypasses the assignDataColorMapping grey override entirely (because isCustomColor becomes true and the single-series shortcut is skipped). Multi-series charts also benefit — they now use moderate intensity across the palette, giving stronger visual contrast than the default faint fallback.
