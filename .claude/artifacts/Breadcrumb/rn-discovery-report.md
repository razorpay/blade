## Unsupported CSS

The web implementations do NOT use any Styled components (no `Styled*.web.tsx` files). All rendering uses `BaseBox`, `Text`, and `BaseLink` directly — all of which are cross-platform. Therefore:

- No `boxShadow`, `transition`, `cursor`, or pseudo-elements to remove
- No CSS Grid
- No CSS-only animations

## Structural Differences (Web vs Native)

| Web Pattern | Native Equivalent | Action |
|-------------|------------------|--------|
| `BaseBox as="nav"` | `BaseBox` (no `as` prop needed for semantics — use `accessibilityRole`) | Replace `as="nav"` with `makeAccessible({ role: 'navigation' })` on the outer View |
| `BaseBox as="ol"` | `BaseBox` (no semantic list element) | Remove `as="ol"`, keep gap/flex props, add `makeAccessible({ label })` |
| `BaseBox as="li"` | `BaseBox` | Remove `as="li"`, keep flex props |
| `BaseBox as="span"` (separator) | `BaseBox` (no `as`) | Remove `as="span"` |
| `style={listStyleNone}` | N/A | Remove — no CSS list-style in RN |
| `Text as="span"` in Separator | `Text` (no `as`) | Remove `as="span"` |

## View Tree: Breadcrumb (Native)

```
<BaseBox accessibilityRole="navigation" ...metaAttribute ...analyticsAttribute>
  <BreadcrumbContext.Provider value={contextValue}>
    <BaseBox
      display="flex" flexDirection="row" flexWrap="wrap"
      gap="spacing.2" alignItems="center"
      accessibilityLabel={accessibilityLabel}
    >
      {children.map((child, index) => (
        <BaseBox
          key={index}
          display="flex" flexDirection="row"
          alignItems="center" gap="spacing.2"
          ...metaAttribute(BreadcrumbItem)
        >
          {child}
          {/* Separator — no aria-hidden needed in RN */}
          {showSeparator && <Separator size={size} color={color} />}
        </BaseBox>
      ))}
    </BaseBox>
  </BreadcrumbContext.Provider>
</BaseBox>
```

Note: RN flex default is column; must explicitly set `flexDirection="row"` on all horizontal flex containers.

## View Tree: BreadcrumbItem (Native)

```
/* isCurrentPage = true */
<BaseBox display="flex" flexDirection="row" gap="spacing.2" alignItems="center">
  {Icon && <Icon size={size} color={...} />}
  <Text weight="medium" size={size} color={...}>{children}</Text>
</BaseBox>

/* isCurrentPage = false */
<BaseLink
  size={size} color={color} opacity={...}
  icon={Icon} href={href} onClick={onClick}
  accessibilityProps={{ label: accessibilityLabel }}
  {...makeAnalyticsAttribute(rest)}
>
  {children ?? ''}
</BaseLink>
```

## Type Changes Required

`BreadcrumbItemProps.onClick` is typed as `LinkProps['onClick']` which resolves to `(event: SyntheticEvent) => void`. BaseLink already handles this with a `@ts-expect-error` cast on native. No new `Platform.Select` needed in `types.ts` — the existing type is permissive enough.
