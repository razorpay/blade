## Key Implementation Notes

1. The `StyledQuickFilterWrapper` wraps `View` (not `Pressable`) on native because there is no `<label>` equivalent that extends the tap area. The `Radio`/`Checkbox` components inside are the actual press targets and handle their own selection via the `RadioGroup`/`CheckboxGroup` context.

2. The CSS Grid space-reservation trick (invisible bold `Text` clone) is dropped on native. A single `Text` with `weight={isSelected ? 'semibold' : 'medium'}` is rendered directly — RN font-weight changes do not cause container-width layout shifts.

3. No animations are needed. The web `background-color` transition only triggers on hover, which does not exist on touch devices. The `isSelected` background color changes immediately on native.

4. `types.ts` requires no changes — no DOM event types or HTML refs are used.

5. `flexDirection` must be explicitly set to `'row'` in the styled component since RN defaults to `'column'`.

## Decisions

| # | Decision | Choice | Rationale |
|---|----------|--------|-----------|
| 1 | Wrapper element | `View` not `Pressable` | Radio/Checkbox own their press area; a `Pressable` wrapper without `onPress` adds no value and could interfere with inner event bubbling |
| 2 | Space-reservation text trick | Drop on native | RN font-weight changes don't cause layout shift; single Text node is equivalent |
| 3 | Background transition | Remove entirely | Transition is hover-only — no hover on touch; selected state still reflects immediately via `isSelected` |
| 4 | `borderRadius` | `makeBorderSize(theme.border.radius.small)` | Must be a number on native, not a string token |
