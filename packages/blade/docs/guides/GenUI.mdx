# Blade GenUI SDK

Blade GenUI renders structured, streaming AI responses with Blade components. It includes a
small set of schema-driven components and an extensible registry for product-specific UI.

> GenUI is currently available on web only.

## Quick start

Wrap the renderer in `GenUIProvider` and pass the parsed component array to
`GenUISchemaRenderer`.

```tsx
import {
  GenUIProvider,
  GenUISchemaRenderer,
  type GenUIComponent,
} from '@razorpay/blade/components';

const components: GenUIComponent[] = [
  { component: 'TEXT', content: '# Payment summary\n\nHere is the latest status.' },
  { component: 'BADGE', text: 'Settled', color: 'positive' },
];

function PaymentSummary() {
  return (
    <GenUIProvider>
      <GenUISchemaRenderer components={components} />
    </GenUIProvider>
  );
}
```

## Built-in components

| Component    | Purpose                                               |
| ------------ | ----------------------------------------------------- |
| `TEXT`       | Markdown content rendered with Blade typography       |
| `CHART`      | Bar, line, area, and pie charts                       |
| `TABLE`      | Data tables with typed cells and optional row actions |
| `CARD`       | Card container with optional header and footer        |
| `BADGE`      | Status badge                                          |
| `STACK`      | Vertical or horizontal flex layout                    |
| `GRID`       | CSS grid layout                                       |
| `INFO_GROUP` | Key-value information                                 |
| `BUTTON`     | Action button                                         |
| `LINK`       | Inline action link                                    |
| `ALERT`      | Feedback message with optional actions                |
| `AMOUNT`     | Formatted monetary value                              |
| `DIVIDER`    | Horizontal or vertical divider                        |
| `SPACER`     | Explicit spacing between components                   |

The complete property contract for each schema is available through the Blade MCP
`get_blade_component_docs` tool by requesting `GenUI`.

## Custom components

Use `GenUICustomComponent` to define the schema and register its renderer through
`GenUIProvider`.

```tsx
import {
  Amount,
  Card,
  CardBody,
  Skeleton,
  Text,
  GenUIProvider,
  GenUISchemaRenderer,
  type GenUIComponentRenderer,
  type GenUICustomComponent,
} from '@razorpay/blade/components';

type RevenueWidgetSchema = GenUICustomComponent<
  'REVENUE_WIDGET',
  {
    title?: string;
    value?: number;
    currency?: 'INR' | 'MYR';
  }
>;

const RevenueWidget = ({ title, value, currency }: RevenueWidgetSchema) => {
  if (!title || value === undefined) {
    return <Skeleton width="100%" height="96px" />;
  }

  return (
    <Card width="100%">
      <CardBody>
        <Text size="small" color="surface.text.gray.muted">
          {title}
        </Text>
        <Amount value={value} currency={currency === 'MYR' ? 'MYR' : 'INR'} />
      </CardBody>
    </Card>
  );
};

function RevenueResponse() {
  const revenueComponent: RevenueWidgetSchema = {
    component: 'REVENUE_WIDGET',
    title: 'Revenue this month',
    value: 125000,
    currency: 'INR',
  };

  return (
    <GenUIProvider
      config={{
        components: {
          REVENUE_WIDGET: { renderer: RevenueWidget as GenUIComponentRenderer },
        },
      }}
    >
      <GenUISchemaRenderer components={[revenueComponent]} />
    </GenUIProvider>
  );
}
```

## Handling streamed schemas

LLM output can be parsed repeatedly while JSON is still being generated. Components may
therefore receive partial strings, incomplete array entries, or missing optional properties.

- Keep schema properties optional when they can arrive incrementally.
- Render a skeleton or nothing until the minimum required data is present.
- Validate enum-like strings before forwarding them to Blade components.
- Filter incomplete array entries before rendering collections.
- Use optional chaining for nested objects.
- Avoid side effects during render because the same component can update many times per response.

The built-in renderers already follow these rules. Custom renderers are responsible for handling
their own partial state.

## Action handling

`BUTTON`, `LINK`, `ALERT`, and table row actions send their structured action to the callback in
the provider configuration. The consumer decides whether to submit another prompt, navigate, or
perform a product action.

```tsx
import { GenUIProvider, GenUISchemaRenderer } from '@razorpay/blade/components';

function SuggestedAction() {
  return (
    <GenUIProvider
      config={{
        onActionClick: (action) => {
          if (action.type === 'CLICK' && typeof action.data?.message === 'string') {
            console.log(action.data.message);
          }
        },
      }}
    >
      <GenUISchemaRenderer
        components={[
          {
            component: 'BUTTON',
            text: 'Show failed payments',
            action: {
              type: 'CLICK',
              eventName: 'show_failed_payments',
              data: { message: 'Show failed payments from the last seven days' },
            },
          },
        ]}
      />
    </GenUIProvider>
  );
}
```

## Text animation

Set `isAnimating` while the model is streaming markdown. `animateOptions` controls the word
animation used by the `TEXT` renderer.

```tsx
<GenUISchemaRenderer
  components={[{ component: 'TEXT', content: streamedMarkdown }]}
  isAnimating={isStreaming}
  animateOptions={{ duration: 300, sep: 'word' }}
/>
```
