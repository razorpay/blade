# Which selection component should I use?

Several Blade components let a user pick from a set of options — [Switch](/?path=/docs/components-switch--docs), [Checkbox](/?path=/docs/components-checkbox-checkbox--docs), [Radio](/?path=/docs/components-radio-radiogroup--docs), [Chip](/?path=/docs/components-chip-chip--docs), [SegmentedControl](/?path=/docs/components-segmentedcontrol--docs), [Tabs](/?path=/docs/components-tabs--docs) and [Dropdown](/?path=/docs/components-dropdown-dropdown--docs) — and a few of them look visually similar.

The most commonly confused pair is [Tabs](/?path=/docs/components-tabs--docs) and [SegmentedControl](/?path=/docs/components-segmentedcontrol--docs), so this guide helps you pick the right one based on how the selection behaves, not just how it looks.

Use the decision tree below, then refer to the quick-reference table for a summary.

## Tabs vs SegmentedControl

[Tabs](/?path=/docs/components-tabs--docs) and [SegmentedControl](/?path=/docs/components-segmentedcontrol--docs) are the most commonly confused because they look alike, so the distinction is worth calling out — it's about **what the selection does**:

- [Tabs](/?path=/docs/components-tabs--docs) are navigational — each tab reveals its **own, distinct content panel**, so use them to switch between larger sections of content within the same page context (e.g. Subscription / Plans / Settings).
- [SegmentedControl](/?path=/docs/components-segmentedcontrol--docs) is a compact, form-like control that **changes how the _same_ content is displayed or filtered**, and is best for 2–5 short, consistent options (e.g. a Daily / Weekly / Monthly toggle above a single chart, or a List / Grid view switch).

If selecting an option swaps out an entire panel of content, reach for [Tabs](/?path=/docs/components-tabs--docs); if it just re-filters or re-represents content that's already on the page, reach for [SegmentedControl](/?path=/docs/components-segmentedcontrol--docs).
